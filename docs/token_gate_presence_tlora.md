# SDXL Presence Token Gate (T-LoRA) 機能ガイド

## 概要
Presence モード T-LoRA は、特定のキャラクタートークンがプロンプトに含まれるときだけ LoRA を有効化し、キャラ名が無い場合や負例ステップでは自動的に無効化する学習モードです。SDXL のクロスアテンションヘッドごとにシグモイドゲート値 \(g\) を学習し、\(g\) を LoRA 出力に掛け合わせることでキャラ名の有無に応じた通電/遮断を実現します。アンカー蒸留・負ビット学習と組み合わせることで、「名前ありでも LoRA を切りたいケース」を矛盾なく学習させられます。

## 仕組みのポイント
- **ゲート入力**: 指定したキャラクタートークンの出現フラグ \(z_{1:T}\) と、オプションの負ビット \(z_{\text{neg}}\) をまとめてヘッドごとの線形層に入力します。
- **ゲート出力**: \(g = \sigma(b + \mathbf{w}\cdot\mathbf{z})\) を計算し、LoRA のクロスアテンション出力をヘッド単位でスケーリングします。
- **ステップ種別**:
  - *正例* (pos): 通常の拡散損失。指定確率でキャラ名のみドロップアウト。
  - *アンカー* (anchor): キャラ名を必ず除去して LoRA 無効時の UNet 出力に蒸留。
  - *負ビット* (negbit): キャラ名は残したまま内部で \(z_{\text{neg}}=1\) とし、LoRA 無効の挙動を蒸留。
- **ログ監視**: `g_mean_pos` / `g_mean_anchor` / `g_mean_negbit` が自動で記録され、正例で高く・負例で低くなるほど想定通りに収束しています。

### ステップ種別の役割をもう少し詳しく
- **正例 (pos)** は「キャラ名があるなら LoRA を通電する」動作を覚えさせるステップです。`token_drop_prob` によりキャラ名だけがたまに欠ける版も混ぜ、ゲートのオン/オフ境界を自然に学習させます。損失は通常のノイズ予測で、LoRA 出力をそのまま使用します。
- **アンカー (anchor)** は「キャラ名が無ければ LoRA を切る」挙動を学習する専用ステップです。キャラ名は必ず取り除き、LoRA を無効化した UNet の出力に蒸留します。正例との違いは、キャラ名を消した後に **LoRA 無効の挙動を教師にする** 点です。
- **負ビット (negbit)** はキャラ名を残したまま内部フラグ `z_neg` を立てることで、「キャラ名はあるが特定条件では LoRA を切る」ケースを学習させます。テキスト側に負例を混ぜなくても矛盾なく OFF 挙動を覚えさせられるのが特徴です。

## コマンドラインオプション
| オプション | 既定値 | 説明 | 推奨値の目安 |
| --- | --- | --- | --- |
| `--token_gate` | `False` | Presence T-LoRA を有効化します。 | 利用時は必須。
| `--token_gate_tokens` | – | ゲート対象となるキャラ名（カンマ区切り、生テキスト検索）。大文字小文字は無視されます。 | 少なくとも 1 件。主要キャラのみを列挙。
| `--token_gate_scope` | `cross_attn` | ゲートを適用する U-Net モジュール範囲。現在はクロスアテンションヘッド固定。 | デフォルトのまま。
| `--token_gate_dim` | `head` | ゲートの粒度。現在はヘッド単位のみ。 | デフォルト固定。
| `--token_gate_l1` | `0.0` | ゲート重みの L1 正則化係数。大きくすると LoRA が切れやすくなります。 | `1e-4` 〜 `5e-4` で調整。
| `--token_drop_prob` | `0.0` | 正例ステップでキャラ名のみをドロップする確率。LoRA のオンオフ境界を学習しやすくします。 | `0.1` 〜 `0.2` を推奨。多すぎるとキャラ性能が落ちやすい。
| `--anchor_ratio` | `0.0` | アンカー蒸留ステップの比率。キャラ名なし＝LoRA 無効動作を覚えさせます。 | `0.25` 〜 `0.35`。学習時間と相談して調整。
| `--neg_gate_bit` | `False` | 内部の負ビット \(z_{\text{neg}}\) を有効化。負ビット比率を使う場合は必須。 | `--neg_gate_ratio` を使うなら必須。
| `--neg_gate_ratio` | `0.0` | 負ビットステップの比率。キャラ名ありで LoRA を切る挙動を蒸留します。 | `0.08` 〜 `0.12`。`--neg_gate_bit` とセットで使用。
| `--anchor_loss_weight` | `1.0` | アンカー/負ビット蒸留の損失重み。大きくすると LoRA 無効時の忠実度を優先。 | まずは `1.0`。必要に応じ `0.7`〜`1.3`で微調整。

#### `--token_gate_l1` の意味と調整のコツ
ゲート重みに掛かる L1 正則化係数です。値を大きくすると重みが 0 に引き寄せられ、シグモイド出力 `g` が 0.5 付近に留まりやすくなる＝LoRA を切る方向へバイアスがかかります。逆に小さくすると重みが自由に伸び、`g` が 0/1 付近まで振り切れて ON/OFF のコントラストが強まる傾向があります。最初は `1e-4` 前後から始め、ゲートが開きっぱなし（`g_mean_pos` が高いのに anchor/negbit でも高い）なら係数を上げ、逆に閉じすぎてキャラが薄い時は係数を下げてください。

#### `--anchor_loss_weight` の意味と調整のコツ
アンカー／負ビットで行うベース UNet への蒸留損失に掛ける重みです。値を上げると「LoRA を切ったときはベースの出力に合わせろ」という圧力が強まり、OFF 側の挙動を厳密に覚えます。下げると正例（LoRA ON）の表現を優先しやすく、キャラが薄くなるのを防ぎやすくなります。デフォルトは `1.0`。OFF が弱い場合は 1.2〜1.3 程度に上げ、キャラが出にくくなったら 0.8 前後に下げる、といった調整が目安です。

### 併用時の注意
- `--cache_text_encoder_outputs` とは同時使用できません（生テキストに対するキャラ名検出が必要なため）。
- 指定トークンは生キャプションに対して部分一致検索されます。前後のスペースや記号は自動で正規化されませんので、訓練データ側の表記ゆれを確認してください。
- アンカー/負ビット比率の合計は 1 以下である必要があります。比率が高いほど正例ステップが減る点に注意してください。

## はじめて試すための推奨プリセット
下記は SDXL LoRA を presence モードで初期検証したい場合の一例です。`キャラA,キャラB` は実際に学習したいキャラ名へ置き換えてください。

```bash
--token_gate \
--token_gate_tokens "キャラA,キャラB" \
--token_gate_l1 1e-4 \
--token_drop_prob 0.15 \
--anchor_ratio 0.3 \
--neg_gate_bit \
--neg_gate_ratio 0.1 \
--anchor_loss_weight 1.0
```

### 推奨モニタリング
- TensorBoard / WandB 等で `g_mean_pos` が 0.7〜0.9 付近、`g_mean_anchor` と `g_mean_negbit` が 0.1〜0.3 付近を目安にすると挙動の良し悪しを判別しやすいです。
- `loss/current` が過度に増加する場合は `--anchor_loss_weight` を 0.8 程度に下げるか、`--token_gate_l1` を弱めてください。

### 調整のヒント
- キャラが薄くなる場合: `--token_drop_prob` を下げる、あるいはアンカー/負ビット比率を減らす。
- LoRA が切れにくい場合: `--token_gate_l1` を強める、`--neg_gate_ratio` を少し増やす。
- 学習が不安定な場合: バッチサイズ 1（Grad Accum で調整）と `fp16_safe_norms` を併用し、勾配スキップ設定を活用してください。

これらの設定を起点に、データセット特性（キャラ名の頻度、負例の有無）に合わせて微調整してみてください。

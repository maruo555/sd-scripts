# Self-Mask LoRA オプション解説

Self-Mask LoRA（自己注意マスク更新）で使用する CLI フラグの意味と、実運用での目安値をまとめます。すべてのフラグは `sdxl_train_network.py` など SDXL 用トレーナーでのみ有効です。未指定の場合は従来挙動（マスクなし）となります。

## 基本スイッチ

| フラグ | 説明 | 推奨初期値 |
| --- | --- | --- |
| `--self_mask_enable` | 自己マスク機能のオン／オフ。指定時のみ以下のオプションが評価されます。 | 最初に試す際は必ず付与 |

## トークン選択関連

| フラグ | 説明 | 目安 |
| --- | --- | --- |
| `--self_mask_tokens` | 前景として強調したいトークンと重みをカンマ区切りで列挙します。`PATTERN[:WEIGHT]` 形式で、ワイルドカード（`*` や `?`）が利用可能です。例: `"CHAR*:1.0,OUTFIT*:0.4"` | 主要キャラタグは 1.0、衣装など部分的に強調したいタグは 0.3〜0.5 |
| `--self_mask_ignore_tokens` | 背景扱いにしたくない（無視したい）タグを指定します。形式は `--self_mask_tokens` と同じです。ここに入ったタグは前景にも背景にも使われません。 | 捨てタグや汎用タグを列挙。例: `"BADTAG*,BG_*"` |

### 補足

- 前景に列挙されなかったトークンは自動的に背景候補となり、`self_mask_other_scale` 等で扱われます。
- ワイルドカードは `fnmatch` 互換です。`*` は任意長、`?` は 1 文字、`[abc]` は候補セットを表します。
- トークン列はキャプション・トークン化の結果と照合されるため、サブワード分割された場合も自動集約されます。

## U-Net からの特徴取得

| フラグ | 説明 | 目安 |
| --- | --- | --- |
| `--self_mask_layers` | マスク生成に使用する U-Net ブロックを指定します。`mid`（ミドルブロック）、`up`（デコーダ側）、`down`（エンコーダ側）がショートカットで、モジュール名を直接書くことも可能です。 | まずは `mid` のみが安定。必要に応じて `mid,up` のように追加 |

## マスク生成モード

| フラグ | 説明 | 目安 |
| --- | --- | --- |
| `--self_mask_mode` | 前景マップの混合方法。`ratio`（前景 / (前景 + 背景)）、`diff`（前景 − 背景）、`contrast`（前景アテンションと強制ゼロ化したコントラストパスの差分）。 | 初回は `ratio`。差分が弱い場合は `diff`、過学習気味なら `contrast` |
| `--self_mask_other_scale` | 背景アテンションの寄与係数。`ratio/diff` モードで背景を弱め／強める際に調整します。 | 1.0（デフォルト）。背景影響をさらに抑えたい場合 0.5 程度 |

## マスク二値化・平滑

| フラグ | 説明 | 目安 |
| --- | --- | --- |
| `--self_mask_thresh` | マスクを 0/1 に落とす閾値。1 以下なら値そのもの、1 より大きい場合は上位パーセント（例: 60 → 上位 60%）。 | `0.6` 付近から調整。カバー率が低い場合は 0.5、狭めたい場合は 0.7 以上 |
| `--self_mask_smooth` | 形態学的開閉やガウシアンぼかしを指定。`open=1,blur=1` のようにカンマ区切りで指定します。`open`/`close` は回数、`blur` は半径相当。 | まずは `open=1,blur=1`。輪郭が荒い場合は `close=1` 追加 |
| `--self_mask_ema_decay` | マスクの指数移動平均 (EMA) を有効化し、信頼度が低いステップでフォールバックに使用します。0〜1。 | 0.9 前後で安定化。不要なら未指定 |

## ウォームアップ・信頼度制御

| フラグ | 説明 | 目安 |
| --- | --- | --- |
| `--self_mask_warmup` | 学習序盤にマスクを無効化する時間割合。学習が安定するまで待ちたい場合に指定。 | 0.1（総ステップの 10%）程度 |
| `--self_mask_conf_min_sep` | 前景／背景の平均アテンション分離度がこの値未満ならマスクを適用しません。 | 0.02〜0.05。数値が小さいほどマスクが早く有効化 |
| `--self_mask_conf_cov_range` | マスクの被覆率（0〜1）が許容レンジ外の場合は EMA フォールバック／無効化します。`min,max` 形式。 | `0.05,0.45` など。被りすぎ／少なすぎを抑止 |

## 重み付け・ログ関連

| フラグ | 説明 | 目安 |
| --- | --- | --- |
| `--self_mask_fg` | 前景画素に掛ける損失係数。 | 1.0〜1.2 |
| `--self_mask_bg` | 背景画素の損失係数。0 にはできません。 | 0.2〜0.4 |
| `--self_mask_cpu_offload` | 収集したアテンション統計をステップ間でCPUに退避し、VRAM使用量を抑えます。 | VRAMが逼迫するときに有効化 |
| `--self_mask_log` | 指定ステップ毎にログへ分離度・被覆率などを出力。0 で無効。 | 100〜200 ステップごとがお勧め |
| `--self_mask_log_mask_dir` | マスク画像を保存するディレクトリ。指定すると PNG で保存します。 | 調査したい場合にパスを指定 |
| `--self_mask_log_mask_interval` | マスク画像を保存する間隔。`--self_mask_log_mask_dir` とセットで使用。 | 500〜1000 ステップ程度 |

### VRAM と動的マスクの注意点

- Self-Mask は全ステップで U-Net のクロスアテンションをフックし、前景／背景アテンションを集計します。そのため **マスクなし学習より 2〜4GB 程度 VRAM が増える** 場合があります（バッチサイズ・解像度・対象レイヤー数によって変動）。
- `--self_mask_cpu_offload` は集計済みアテンションを CPU に退避して VRAM 常駐量を抑えますが、GPU 上で計算される注意スコア自体は生成されるため、ピーク VRAM を完全には削れません。VRAM が逼迫する環境では、バッチサイズや `--self_mask_layers` を減らすなどの調整も併用してください。

### 手動マスクとの違い

- Self-Mask が生成するのは **latent 空間上の低解像度マスク** で、キャプションとモデルの注意分布に応じて毎ステップ動的に変化します。学習が進むとマスクも変わるため、固定ファイルとして保存して使い回す設計ではありません。
- 手動で用意した 2D マスク（画像マスク）は高解像度かつ固定の領域を指定できます。正解マスクが用意できるデータセットでは従来のマスク学習の方が確実な場合もあります。
- Self-Mask は「マスク画像を作る余力がない」「キャプション内容に合わせて自動で被写体を抽出したい」といったケースに向いています。用途に応じて固定マスクと使い分けてください。

## 参考プリセット

```bash
--self_mask_enable \
--self_mask_tokens "CHAR*:1.0,OUTFIT*:0.3" \
--self_mask_ignore_tokens "BG_*,THROW*" \
--self_mask_layers "mid" \
--self_mask_mode ratio \
--self_mask_thresh 0.6 \
--self_mask_fg 1.0 --self_mask_bg 0.3 \
--self_mask_warmup 0.1 \
--self_mask_conf_min_sep 0.03 \
--self_mask_smooth "open=1,blur=1" \
--self_mask_log 100
```

この設定で分離度や被覆率のログを確認し、マスクが狭すぎる場合は `--self_mask_thresh` を下げる／`--self_mask_conf_cov_range` を外して様子を見る、背景が強すぎる場合は `--self_mask_bg` をさらに下げる、といった調整を行うと安定します。
